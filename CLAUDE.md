# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Project Overview

This is a Claude Code Skill (`perfil-rrhh-pdf-word-pdf`) for processing HR/CV documents. It takes a PDF resume, generates two professionally formatted Word documents from opposite HR perspectives (corporate vs. achievement-focused), evaluates them with a rubric, and exports the best version as a PDF.

## Setup

Install Python dependencies from within the skill directory:

```bash
cd .claude/skills/perfil-rrhh-pdf-word-pdf/
pip install -r requirements.txt
```

For scanned PDFs, install Tesseract + ocrmypdf: `brew install tesseract ocrmypdf`

Prepare the output folder:

```bash
bash .claude/skills/perfil-rrhh-pdf-word-pdf/scripts/prepare_output.sh
```

## Running the Scripts Manually

All scripts run from the repo root. Output always goes to `output/` at the repo root.

```bash
# 1. Extract text from PDF
python .claude/skills/perfil-rrhh-pdf-word-pdf/scripts/extract_pdf_text.py "/path/to/cv.pdf" -o output/extracted.txt

# 2. Build Word docs from JSON payloads (generated by Claude)
python .claude/skills/perfil-rrhh-pdf-word-pdf/scripts/build_docx.py --json output/payload_v1.json --out output/perfil_v1_rrhh_corporativo.docx
python .claude/skills/perfil-rrhh-pdf-word-pdf/scripts/build_docx.py --json output/payload_v2.json --out output/perfil_v2_rrhh_logros.docx

# 3. Convert chosen Word doc to PDF
python .claude/skills/perfil-rrhh-pdf-word-pdf/scripts/docx_to_pdf.py output/perfil_v1_rrhh_corporativo.docx -o output/perfil_rrhh_final.pdf
```

## Architecture

**Data flow:**

```
User PDF → extract_pdf_text.py → raw text
         → Claude generates 2 JSON payloads (v1: corporate theme, v2: executive theme)
         → build_docx.py → two .docx files
         → Claude evaluates with HR rubric → selects best
         → docx_to_pdf.py → final PDF
```

**Key scripts:**

- `extract_pdf_text.py` — uses `pdfplumber`; warns if pages are empty (scanned PDF needs OCR)
- `build_docx.py` — accepts structured JSON; applies color theme (corporate=blue, executive=teal/gray); Calibri 11pt, custom spacing
- `docx_to_pdf.py` — tries LibreOffice (`soffice`) first, falls back to `docx2pdf` (requires `--word` flag or Word installed)
- `prepare_output.sh` — clears `output/` and returns its absolute path

**JSON payload schema** (for `build_docx.py`): see `SKILL.md` for the full schema. Key fields: `title`, `theme` (`"corporate"` or `"executive"`), `hr_analysis` block, `sections` array.

**HR evaluation rubric** (5 criteria): clarity, adequacy, differentiation, length, keywords — detailed in `SKILL.md`; extended criteria in `reference.md`.
